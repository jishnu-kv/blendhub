Frequently Asked Questions

Detailed metrics regarding licensing, offline file directories, and startup environments checks.

BlendHub scans your default Blender addon directories and registers Installed & Global Addons. The central Addon Manager detects which plugins are compatible with each of your installed Blender executable files. You can trigger synchronization, copying configurations, extension items, or presets from one version (e.g. 5.1) directly to another (e.g. 4.5) with a single click.

Your privacy is paramount. BlendHub is designed to be 100% native and local. Files, paths, local project notes, backups, and configurations reside exclusively on your Windows drive. It optionally checks extensions.blender.org via direct network connections only to query addon catalog updates and download packages.

BlendHub registers projects dynamically and acts as a binder. By pinning a directory as a project, BlendHub automatically exposes its folder files tree, parses direct link associations (like materials and GLB mockups), supports quick creation of task cards and annotations, and launches the project file directly in the accurate, auto-detected Blender release version.

Absolutely. In the Settings tab, you can register custom portable Blender zip extractions or directories in custom locations. BlendHub intercepts the version tag and adds it cleanly to your Launchers panel instantly.
